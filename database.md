# StadiumMind AI - Firestore Database Design

StadiumMind AI uses Google Cloud Firestore for real-time document storage. Its hierarchical and flexible document structure enables low-latency streaming of sensor data, passenger requests, incidents, and volunteer dispatches directly to client applications.

---

## 1. Collection Structure & Schemas

### `Users` Collection
Stores user profiles for fans, staff, and VIP guests.
```json
{
  "userId": "string (UUID)",
  "fullName": "string",
  "email": "string",
  "role": "fan | volunteer | organizer | responder | vip",
  "preferredLanguage": "string (ISO 639-1)",
  "accessibilityProfile": {
    "wheelchairRequired": "boolean",
    "visualAssistance": "boolean",
    "auditoryAssistance": "boolean",
    "sensorySensitive": "boolean"
  },
  "currentLocation": {
    "latitude": "number",
    "longitude": "number",
    "accuracy": "number",
    "lastUpdated": "timestamp"
  },
  "assignedTicketId": "string (Ticket Document Ref)",
  "createdAt": "timestamp"
}
```

## 2. Indexes & Performance Optimization

Firestore requires composite indexes for complex filtering. Below are the mandatory indexes defined in `firestore.indexes.json` for StadiumMind AI:

1. **Incidents Search**:
   - `category` ASC, `status` ASC, `createdAt` DESC
   - Allows operations staff to filter reported safety events by status and view the most recent first.
2. **Volunteer Search**:
   - `status` ASC, `skills` ARRAY_CONTAINS, `rating` DESC
   - Enables the Dispatch Engine to fetch available volunteers holding a specific skill (e.g., first aid) sorted by performance rating.
3. **Sensor Query**:
   - `sensorId` ASC, `timestamp` DESC
   - Allows fetching the latest history of a specific IoT node.

### Cost Control Strategy
To avoid massive Firestore reads/writes, IoT raw telemetry runs through Cloud Dataflow, which performs **sliding window aggregations** (e.g., 5-minute averages) and writes only the summary telemetry to `CrowdZones` and `Predictions`. The raw high-frequency `SensorData` collection is backed up to Google Cloud Storage as cold logs and streamed to BigQuery for analytical query executions.
